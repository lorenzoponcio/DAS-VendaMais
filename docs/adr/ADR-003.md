# ADR - Comparação de Bibliotecas para Acesso ao Banco de Dados

## Status

Aprovado

## Contexto

Foi realizada uma Prova de Conceito (PoC) para comparar empiricamente o desempenho de duas abordagens de acesso ao banco de dados SQL Server em uma Azure Function:

* Driver nativo: `pymssql`
* ORM/Abstração: `SQLAlchemy`

A comparação foi realizada utilizando:

* Mesmo banco de dados
* Mesma tabela (`erp.pedido_item`)
* Mesma consulta SQL:

```sql
SELECT TOP 5 * FROM erp.pedido_item
```

O objetivo foi medir o tempo necessário para consultar e trazer os registros para memória.

| SQLAlchemy | ORM           | 0.0194 s    | 0.8067 s    |

---

## Análise

Os resultados demonstram que ambas as bibliotecas apresentaram desempenho semelhante na execução da consulta SQL, com diferença média de aproximadamente 1,6 milissegundos.

Entretanto, ao considerar o tempo total da operação, incluindo criação de conexão, execução da consulta e retorno dos dados, o SQLAlchemy apresentou um overhead significativamente maior.

O tempo total médio do SQLAlchemy foi aproximadamente 2,4 vezes superior ao do pymssql.

Essa diferença é esperada devido à camada adicional de abstração fornecida pelo SQLAlchemy.

---

## Decisão

Para operações simples de leitura com foco em desempenho, a biblioteca `pymssql` apresentou o melhor resultado nesta PoC.

---

## Consequências

### Positivas

* Menor tempo total de execução.
* Menor overhead.
* Implementação simples para consultas diretas.

### Negativas

* Menor abstração.
* Menor flexibilidade para modelagem complexa.
* Maior acoplamento às consultas SQL.

### Considerações sobre SQLAlchemy

Apesar do desempenho inferior nesta PoC, o SQLAlchemy continua sendo uma alternativa válida para cenários que priorizam manutenção, reutilização de código e abstração de acesso aos dados.
