# ADR-002: Escolha do Azure SQL Database como repositório analítico estruturado da solução

**Status:** [Aceito]  |  **Data:** [2026-04-06]  |  **Autores:** [Carlos Augusto Brazão da Silva,Juliana Seemann, Lorenzo Poncio, Matheus Henrique Faria Motta]

## Contexto

A plataforma da VendaMais precisa armazenar os dados extraídos do ERP de forma organizada, com rastreabilidade e histórico, possibilitando transformações, análises e consumo por dashboards no Power BI.

A arquitetura definida contempla duas camadas de armazenamento que se complementam: o Azure Blob Storage para dados brutos e o Azure SQL Database para dados processados e normalizados, funcionando como a principal fonte de dados para relatórios.

## Os principais requisitos do projeto são:

- Armazenamento estruturado em ambiente de nuvem
- Capacidade de rastrear e manter histórico dos dados
- Disponibilização de tabelas analíticas confiáveis e bem organizadas
- Atualização diária dos dashboards
- Facilidade de manutenção pela equipe no longo prazo

## Decisão

Optou-se pela utilização do Azure SQL Database como o repositório central para dados tratados e estruturados, enquanto o Azure Blob Storage será mantido como suporte para armazenamento de dados brutos (raw).

Essa escolha se justifica pelo modelo relacional do Azure SQL, que favorece consultas analíticas mais performáticas e integração nativa com o Power BI, atendendo melhor às demandas de relatórios e indicadores (KPIs).

Além disso, em comparação com outras opções, um banco de dados relacional gerenciado reduz a complexidade operacional (quando comparado a máquinas virtuais) e oferece maior adequação para análises estruturadas (em relação ao Blob Storage e bancos NoSQL).

### Na prática, a divisão será:

Azure Blob Storage: armazenamento de dados brutos e históricos
Azure SQL Database: disponibilização de dados estruturados para análise

## (+) Vantagens:

- (+) Simplifica a realização de análises e a integração com o Power BI  ;
- (+) Aumenta a organização, consistência e governança dos dados  ;
- (+) Estabelece uma fonte única e confiável para indicadores  ;
- (+) Garante rastreabilidade com o apoio do Blob Storage  ;
- (+) Possibilita o reprocessamento dos dados quando necessário  ;

## (-) Desvantagens

- (-) Maior custo em comparação com soluções baseadas apenas em arquivos  ;
- (-) Exige modelagem relacional e maior esforço inicial de implementação  ;
- (-) Dependência de serviços gerenciados da Azure (vendor lock-in);

## Alternativas rejeitadas

### Uso exclusivo de Azure Blob Storage:

Não foi adotado por não oferecer suporte adequado a consultas analíticas complexas nem à estruturação necessária para dashboards.

### Banco de dados em máquina virtual (VM):

Descartado devido à necessidade de gerenciar infraestrutura, backups e manutenção manual.

### Uso de banco NoSQL como principal:

Não escolhido por não atender de forma eficiente às necessidades de análises relacionais e consultas estruturadas exigidas pelos KPIs.
